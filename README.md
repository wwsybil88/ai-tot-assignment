# ai-tot-assignment Tree of Thoughts (ToT) 项目实践

本仓库用于探索和实现思维树（Tree of Thoughts）技术。ToT 通过构建多分支思维路径，显著提升了 LLM 在复杂决策任务中的表现。

## 思考题：哪些情况下适合使用 ToT？

在实际应用中，并非所有任务都需要用到 ToT。以下是适合采用 ToT 技术的核心场景：

### 1. 存在明确约束的组合优化问题
当问题有严格的数学或逻辑规则，且需要从大量组合中寻找解时。
* **典型案例：** “24点游戏”、数独（Sudoku）、物流路径规划。
* **原因：** 每一推导步骤都可以被量化评估，模型发现路径错误时可以及时“回溯”。

### 2. 需要多阶段规划的创意写作
传统的流式生成容易在长篇写作中偏离主题。
* **典型案例：** 撰写包含复杂伏笔的短篇小说、多章节的技术方案设计。
* **原因：** ToT 允许模型先生成多个大纲分支，评估哪个分支最具文学价值或逻辑严密性，再深入展开。

### 3. 搜索空间巨大的策略性博弈
在每一步都有多种选择，且每种选择会导致完全不同结局的场景。
* **典型案例：** 国际象棋残局分析、复杂合同条款的风险博弈、迷你填字游戏。
* **原因：** 模型可以模拟不同选择的后果，并根据全局目标（胜率或覆盖率）进行路径优选。

### 4. 高容错成本的任务
如果一次性生成的错误答案会带来严重后果，则需要多路径校验。
* **典型案例：** 科学实验方案设计、复杂的代码重构架构设计。
* **原因：** 通过多分支对比和自我审视，ToT 能显著降低模型产生幻觉（Hallucination）的风险。

---

## 总结：ToT vs CoT 的选择原则
- 如果任务是**“快思考”**（直觉逻辑、简单问答），请使用 **CoT (Chain of Thought)** 以节省成本。
- 如果任务是**“慢思考”**（需要反复权衡、试错、寻找最优解），**ToT** 是目前的最佳选择。
